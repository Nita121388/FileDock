# 2026-03-10

## P15 snapshot retention policy buckets

- Extended snapshot pruning beyond `keep-last` / `keep-days` so FileDock can now keep one newest snapshot per recent UTC day (`keep-daily`) and per recent ISO week (`keep-weekly`).
- Updated the shared prune request schema plus CLI flags, then taught the server retention selector to union all keep rules per device group before deleting old snapshot metadata/manifests.
- Added focused server unit tests covering the new day/week buckets and the union behavior between multiple keep rules.
- Refreshed retention/API/roadmap/script docs so the new policy options are discoverable from both docs and helper scripts.

## P16 agent/daemon scheduling (M2 plan)

- Drafted a concrete M2 proposal for agent scheduling: keep the existing long-running daemon mode as default, and add an opt-in scheduled-runner mode that uses OS-native schedulers to run a new `agent run-once` command periodically.
- Added `docs/agent-scheduling-m2.md` describing goals, CLI surface changes (`agent run-once`, `agent install --mode`), per-platform wiring (systemd timers / launchd StartInterval / Windows Task Scheduler), and acceptance/verification checklists.
- Updated `docs/roadmap.md` to list scheduling M2 as a near-term optional improvement.

## P17 agent/daemon scheduling (M2 implementation)

- Added `filedock agent run-once --config <path>`: runs exactly one snapshot (plus optional start/end heartbeats) and exits.
- Extended `filedock agent install` with `--mode daemon|scheduled` (default `daemon`):
  - `daemon`: keeps a long-running agent alive (cadence is owned by `interval_secs` in the profile TOML).
  - `scheduled`: installs an OS-native scheduler definition that periodically runs `agent run-once`.
- Implemented per-platform wiring (best-effort):
  - Linux: systemd user timer + oneshot unit; status reports the timer enabled/active state; uninstall removes both timer + service.
  - macOS: LaunchAgents `StartInterval` runner (no `KeepAlive`).
  - Windows: Task Scheduler periodic task (whole-minute interval derived from `interval_secs`).
- Updated agent docs (`docs/agent.md`) and added focused unit tests for the new service/timer render templates.
