# 2026-03-11

## P27 ignore rules: nested .gitignore support

- Extended `--respect-gitignore` to respect `.gitignore` files throughout the folder tree (not just the root).
- Implemented lazy per-directory `.gitignore` parsing with caching and correct traversal semantics:
  - `.gitignore` files inside ignored directories are not consulted (matches git behavior).
  - Status ignore reasons now include which `.gitignore` file + pattern matched.
- Updated CLI help + docs to remove the previous root-only scope note, and added focused unit tests covering nested whitelist + ignored-directory behavior.
