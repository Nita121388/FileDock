# Contributing

## Principles

- Prefer small, reviewable commits.
- Each milestone/phase should ship with docs + progress notes.
- Avoid breaking changes without a migration plan.

## Commit Style

Use conventional-ish prefixes:
- `chore:` tooling, repo structure, no behavior change
- `docs:` documentation only
- `feat:` new user-visible feature
- `fix:` bug fix
- `refactor:` internal changes
- `test:` tests only

## Progress Logs

Add a short entry to `progress/YYYY-MM-DD.md` when you make meaningful changes.
